# Concerns & Technical Debt

> Generated by GSD codebase mapper on 2026-01-02

## Identified Concerns

### 1. Complexity Growth
**Risk Level:** Medium

The codebase has grown significantly with multiple product modules:
- `hushh-ai`, `hushh-agent`, `kai`, `kai-india`, `hushh-studio`, `hushh-intelligence`

**Potential Issues:**
- Code duplication across modules
- Inconsistent patterns between modules
- Difficulty maintaining all modules

**Recommendations:**
- Extract shared utilities to common packages
- Establish clear module boundaries
- Consider micro-frontend architecture for larger modules

### 2. UI Framework Mixing
**Risk Level:** Low-Medium

Multiple UI approaches in use:
- Chakra UI (primary)
- Ant Design (secondary)
- Tailwind CSS (utilities)
- Raw CSS

**Potential Issues:**
- Inconsistent styling
- Bundle size bloat
- Learning curve for new developers

**Recommendations:**
- Standardize on Chakra UI for new components
- Use Tailwind only for utility classes
- Phase out Ant Design usage in new code

### 3. Edge Functions Scaling
**Risk Level:** Medium

30+ edge functions with growing complexity:
- Some functions exceed recommended size
- Cold start times may increase
- Deployment complexity

**Potential Issues:**
- Timeout issues for complex operations
- Memory limits for AI operations
- Difficulty debugging in production

**Recommendations:**
- Monitor function execution times
- Consider Cloud Run for heavy operations
- Implement function composition patterns

### 4. Authentication Fragmentation
**Risk Level:** Low

Multiple auth systems:
- Supabase Auth (primary)
- Firebase Auth (phone only)
- Social providers

**Potential Issues:**
- Session management complexity
- Harder to maintain security
- User confusion with multiple login methods

**Recommendations:**
- Document auth flow clearly
- Consider consolidating to Supabase for all auth
- Ensure consistent session handling

### 5. Database Schema Complexity
**Risk Level:** Medium

50+ migration files indicate:
- Rapidly evolving schema
- Multiple feature additions
- Potential for orphaned columns/tables

**Potential Issues:**
- Query performance degradation
- Harder to understand data model
- Migration conflicts

**Recommendations:**
- Document database schema
- Regular schema audits
- Consider schema visualization tools

### 6. AI Provider Dependencies
**Risk Level:** Medium

Heavy reliance on:
- Google Gemini (primary)
- OpenAI (secondary)

**Potential Issues:**
- API rate limits
- Cost management
- Provider outages affect functionality

**Recommendations:**
- Implement fallback providers
- Cache AI responses where appropriate
- Monitor API usage and costs
- Consider self-hosted models for some use cases
