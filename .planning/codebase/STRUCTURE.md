## Key Patterns

### Page Organization
- Each major feature has its own directory under `src/pages/`
- Onboarding uses numbered steps: `Step1.tsx` through `Step15.tsx`
- Index files export from feature directories

### Module Structure
- Self-contained modules: `hushh-ai`, `hushh-agent`, `kai`, etc.
- Each has its own App.tsx, routes, services
- Clean architecture in `hushh-ai` (core/data/domain/presentation)

### Edge Functions
- Each function has its own directory
- `deno.json` for Deno configuration
- `index.ts` as entry point
- Shared code in `_shared/`
