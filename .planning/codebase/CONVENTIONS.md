# Conventions

> Generated by GSD codebase mapper on 2026-01-02

## Code Style

### General Principles (from .cursorrules)
- Write clean, simple, readable code
- Implement features in the simplest possible way
- Keep files small and focused (<200 lines preferred)
- Use clear, consistent naming
- Follow DRY (Don't Repeat Yourself) principles

### TypeScript
- Use TypeScript for all new code
- Prefer interfaces over types
- Use `const` for functions: `const toggle = () => {}`
- Use early returns for error conditions
- Avoid unnecessary curly braces in single-line conditionals

### Naming Conventions

| Type | Convention | Example |
|------|------------|---------|
| Components | PascalCase | `InvestorProfileForm.tsx` |
| Hooks | camelCase with `use` prefix | `usePhoneAuth.ts` |
| Services | camelCase | `geminiService.ts` |
| Types | PascalCase | `InvestorProfile.ts` |
| Constants | UPPER_SNAKE_CASE | `API_ENDPOINTS` |
| Event handlers | `handle` prefix | `handleClick`, `handleSubmit` |
| Booleans | Auxiliary verbs | `isLoading`, `hasError`, `shouldRefresh` |

### File Naming

| Type | Pattern | Example |
|------|---------|---------|
| Components | PascalCase.tsx | `Hero.tsx` |
| Pages | index.tsx or PascalCase | `index.tsx`, `Step1.tsx` |
| Hooks | useFeature.ts | `useStockQuotes.ts` |
| Services | featureService.ts | `ndaService.ts` |
| Types | feature.ts | `kyc.ts`, `onboarding.ts` |
| Edge Functions | kebab-case | `investor-chat/` |

## Component Patterns

### Functional Components
```tsx
// ✅ Preferred: const with arrow function
const MyComponent = ({ prop1, prop2 }: Props) => {
  // Early return for error/loading states
  if (isLoading) return <Spinner />;
  if (error) return <ErrorDisplay error={error} />;
  
  return (
    <Box>
      {/* Component content */}
    </Box>
  );
};
```

### Hook Organization
```tsx
const MyComponent = () => {
  // 1. External hooks (router, params)
  const navigate = useNavigate();
  
  // 2. State hooks
  const [data, setData] = useState<Data | null>(null);
  
  // 3. Custom hooks
  const { user } = useAuth();
  
  // 4. Effects
  useEffect(() => {
    // Effect logic
  }, [dependencies]);
  
  // 5. Event handlers
  const handleSubmit = () => { ... };
  
  // 6. Render
  return <Box>...</Box>;
};
```

## Service Layer Patterns

### API Service Pattern
```typescript
// src/services/{feature}/index.ts
export const featureService = {
  async getData(id: string): Promise<Data> {
    const { data, error } = await supabase
      .from('table')
      .select('*')
      .eq('id', id)
      .single();
    
    if (error) throw error;
    return data;
  },
};
```

### Edge Function Invocation
```typescript
const { data, error } = await supabase.functions.invoke('function-name', {
  body: { param1, param2 }
});
```

## Error Handling

### Try-Catch Pattern
```typescript
try {
  const result = await riskyOperation();
  return result;
} catch (error) {
  console.error('Operation failed:', error);
  toast.error('Something went wrong');
  throw error; // Re-throw if needed
}
```
